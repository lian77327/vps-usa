Как использовать VPS в США для создания личного VPN

Пошаговый гайд: от заказа сервера до первого подключения

Половина людей, которые покупают vps сервер в сша, делают это ради одной задачи: поднять свой VPN. Сервер сша для впн решает вопрос доступа к заблокированным ресурсам, дает американский IP для стриминговых сервисов и обеспечивает адрес сервера америки для vpn.

Вот пошаговая инструкция. Предполагаю, что вы взяли любой VPS из рейтинга с минимальным тарифом (1 ядро, 512 МБ-1 ГБ RAM, Ubuntu 22.04 или Debian 12).

Шаг 1. Заказ и оплата

Выбираете провайдера, оплачиваете минимальный тариф. Получаете на почту IP-адрес, логин root и пароль. Подключаетесь по SSH:

# Подключение к серверу
ssh root@ваш_ip_адрес

Шаг 2. Обновление системы

# Обновляем пакеты
apt update && apt upgrade -y

Шаг 3. Выбор протокола

В 2026 году голый WireGuard на мобильном интернете из России блокируется системами ТСПУ. Работают протоколы с обфускацией:

VLESS Reality (рекомендую). Для систем глубокой инспекции трафика ваш VPN выглядит как обычный HTTPS-визит на google.com или microsoft.com. Не блокируется. Нужен 1 ГБ RAM.
AmneziaWG. Модифицированный WireGuard, который меняет сигнатуру при каждом подключении. Приложение Amnezia VPN ставит серверную часть автоматически - вы даете ему SSH-доступ, и через две минуты всё работает.
Важно: WireGuard по-прежнему живет на домашней оптике у большинства российских ISP. На мобильном интернете - шансов мало. 512 МБ RAM хватает на 10 устройств. Shadowsocks (AEAD) маскирует трафик, проходит DPI, менее надежен, чем VLESS Reality, но проще в настройке.

Шаг 4. Установка 3X-UI (для VLESS Reality)

# Установка панели 3X-UI
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)

Панель развернется за пару минут. Откройте в браузере http://ваш_ip:2053, зайдите с дефолтным логином/паролем (admin/admin), сразу смените их.

Создайте inbound: протокол VLESS, transport - TCP, security - Reality. В поле dest укажите домен для маскировки (например, www.google.com:443). Сгенерируйте пару ключей прямо в панели. Скопируйте ссылку для подключения.

Шаг 5. Клиент на устройстве

На Android/iOS: v2rayNG или Hiddify. Вставляете ссылку из 3X-UI, подключаетесь.

На Windows/macOS: Hiddify или Nekobox. Тот же принцип - вставили ссылку, нажали Connect.

Шаг 6. Настройка BBR (ускорение TCP)

# Включаем BBR — алгоритм контроля перегрузок от Google
echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf
echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf
sysctl -p

BBR - алгоритм контроля перегрузок от Google. На трансатлантических маршрутах увеличивает скорость передачи данных на 20-40%.

Всё. У вас прокси сервер сша для телеграм, YouTube, Netflix и всего остального. Весь трафик через американский выделенный IP адрес, который принадлежит только вам. Никаких общих VPN-сервисов с тысячами пользователей на одном адресе.

Для тех, кто хочет прокси сервера сша socks5 - настройте Xray с протоколом SOCKS через ту же панель 3X-UI. Добавляете отдельный inbound, выбираете SOCKS, указываете порт. Готово.

Какой выделенный IP адрес я получу?
Каждый VPS из рейтинга получает минимум один выделенный IPv4-адрес из американского пула. Это значит, что GeoIP-сервисы определяют ваш сервер как расположенный в США. Адрес не делится с другими пользователями. Дополнительные IPv4 можно докупить (обычно $2-3/мес за штуку). IPv6-подсеть предоставляется бесплатно у большинства провайдеров. Чистоту адреса (отсутствие в спам-листах) стоит проверять при получении через сервисы типа BlacklistMaster или MXToolbox.
VPS или VDS - в чем разница?
Технически - ни в чем. VPS (Virtual Private Server) и VDS (Virtual Dedicated Server) - два названия одной услуги. Маркетинговое разделение, не техническое. Все провайдеры в рейтинге используют KVM-виртуализацию, которая гарантирует выделенные ресурсы для каждого клиента. Ваш впс сша или вдс сша - это одна и та же виртуальная машина с root-доступом.
Зачем нужен облачный сервер в США, если есть обычный VPN-сервис?
Три причины. Первая: ваш IP-адрес принадлежит только вам, он не засвечен на тысячах пользователей и не забанен Netflix. Вторая: вы полностью контролируете сервер - можете установить любой протокол, настроить маршрутизацию, поднять несколько туннелей для разных устройств. Третья: коммерческие VPN-сервисы сами по себе блокируются в России, а ваш личный сервер по IP ничем не отличается от обычного хостинга.
